# 프로젝트 컨텍스트 (Claude 자동 로드)

## 규칙
1. **삭제 관련 내용은 무조건 동의를 받아야 한다.**
2. **모든 백테스트 분석은 항상 2분·5분·10분봉 전부로 나눠서 제시한다** (한 분봉만 보여주지 말 것). 분봉마다 비용 내성·약한 해·빈도가 달라 한 개만 보면 판단이 왜곡됨.

---

## 무엇을 하는 프로젝트인가
XAUUSD(골드) 및 FX 메이저의 **볼린저밴드 돌파 + KTR 그리드(물타기) 전략**을 Dukascopy MID 데이터로 백테스트·고도화한다. 주거래는 골드, 향후 MT4 EA를 위해 통화쌍도 검증.

## 전략 핵심 개념
- **KTR**: 세션별 변동성 단위 (아시아=2봉 범위, 유로/미장=1봉 범위). 그리드 간격의 기준.
- **신호**: BB1(종가 SMA20 ±2σ) 돌파 + BB2(시가 SMA4 ±4σ). 표준편차는 **모표준편차(÷N, Pine ta.stdev와 동일)**.
- **진입 2종**: **v1**=돌파 다음 봉 시가 즉시 진입 / **v2**=돌파 후 BB2 반대밴드 터치(풀백) 시 진입. (그 외 규칙은 v1=v2 동일)
- **그리드**: 돌파가 기준 0/−1/−2/−3/−4/−4.5 ×KTR (6차). 손절 −5×KTR.
- **익절(TP)**: 가장 깊은 체결 + 1.5×KTR, 전 포지션 일괄.
- **6차 처리**: 바닥 +1.0KTR 반등 시 탈출, 아니면 −5 풀스톱.
- 세션/DST: 미국 서머타임(3월 2째주 일요일~11월 1째주 일요일). 미장 시작 22:30/23:30 KST.

> **폴더 구조 (2026-06-18 재구성)**: `2020_2026/` → **`src/`** 로 이동. 데이터 CSV는 **`data/`** 로 분리.
> 분석 스크립트는 bare 파일명(경로 없음)을 읽으므로 **반드시 `data/`에서 실행**: `cd data && python ../src/scripts/XX.py`.
> 중간 산출물(signals_·sim_·ktr_·*.json)은 `data/`에 떨어지고, **HTML 리포트/차트는 전부 `result/`로 출력**(2026-06-19부터, 스크립트가 `../result/`에 씀). 데이터는 전부 **합본 단일 파일** `{inst}_{tf}_2010-01-01_2026-06-16.csv` (xauusd/eurusd/gbpusd/usdjpy × 1·2·5·10m). 1차의 `_2020-01-01_2026-06-16` 분할본은 없음(합본에 포함).

## 핵심 발견 (왜 그런지 포함)
- **랏**: 후방가중이 정답(전방 금지). 후방 1·1·2·2·3·4=생존·증거금(13단위), 강후방 1·1·2·3·5·8=평년효율(20단위)이나 추세장 꼬리위험·증거금 큼. 사다리는 "수익이 아니라 위험 모양"을 바꿈(기대 R은 거의 불변).
- **TP**: 1.5가 효율 정점. 1.8은 추세장(2025)에서 엣지 0·MDD 28% 비단조 함정. 2.0은 MDD만↑.
- **최대 체결**: 6차가 위험조정 최적. 얕게 끊으면 트레이드당 R↑이나 손절 잦아 MDD 폭발.
- **비용이 생사 결정**: 손익분기 스프레드 — 인샘플(2020-26) 2m $0.15~0.20 / 5m $0.34~0.37 / 10m $0.50~0.54. **전 기간(2010-26) 재계산은 더 빡빡: 2m $0.10~0.13 / 5m $0.21~0.25 / 10m $0.30~0.36** (가정비용 $0.30과 맞붙음). 10m만 그나마 흑자지만 비용 여유 거의 없음.
- **리스크%**: 풀스톱=자본의 2~3% 권장. 10%면 MDD 61%·경로 90%가 −50%↑ (연속손절 복리). "한 방 손실"이 아니라 "견딜 낙폭"으로 정할 것.
- **필터 전부 무용 (검증완료)**: 꼬리필터·2h추세필터·골든/데드크로스 모두 net R 감소. 돌파는 반전 이벤트라 MA추세가 후행하고, 그리드가 역추세 진입을 되돌림으로 구제하기 때문. **이 전략엔 진입 필터를 더하지 말 것.** (per-signal로 좋아 보이는 패턴 ≠ 필터로 거르면 이득)
- **운용**: 거래의 ~50%만 실제 진입(순차+새벽컷). v1은 엣지가 하루 종일 분포→시간 선택 자유, v2는 런던/저녁 집중. 평균 ~1.7건/일.
- **자본**: 수익률은 자본 크기 무관(%기반). 하한은 랏 최소단위(~$1만~1.5만). 순차라 동시보유 1개 → 증거금 부담 작음.
- **눌림 통계의 생존편향 (검증완료)**: tf_level_chart의 'N번 눌림' 칸은 *결과적으로* 그 깊이에서 멈춘 트레이드만 모은 것이라 "1번 눌리면 1R 100%"는 착시. 실제로 −1을 한 번이라도 찍은 트레이드의 **~12%(2m 13.8 / 5m 12.6 / 10m 12.3%)는 더 깊이 가서 6차 손절로 끝남**. 손실은 사라진 게 아니라 깊은(6차) 칸에 들어가 있을 뿐. 진입 시점엔 1번에서 멈출지 6차까지 갈지 모름. (`50_pullback_outcome.py`)

## 미해결 / 주의
- **정체 그리드**: 드물게 횡보장에서 그리드가 수주~수개월(최대 151일) 열림 → 자본 묶임. **시간 손절(time-stop) 미검증** = 1차의 마지막 열린 항목.
- 비용 $0.30은 낙관적일 수 있음. 실제 $0.40 초과 시 10m도 위태.

## 파일 맵
- `src/scripts/*.py` — 번호순 분석 파이프라인 (02b 신호생성 → 03~04 시뮬/MFE → 14·28·34·45·47 차트/리포트 → 23~44 최적화·민감도 → 49 추세필터 → 50 눌림결과·생존편향 검증). **표준 라이브러리만 사용(pip 불필요)**. **`cd data && python ../src/scripts/XX.py`로 실행**(bare 파일명이라 cwd가 `data/`여야 함).
  - 헬퍼: `verify_gold_2010_2019.py`(골드 무결성 검증), `merge_gold_full.py`(골드 2010-2019+2020-2026 합본).
- `src/fetch_node/` — Dukascopy 수집. `fetch_inst.mjs`(일반화 페처: `<from> <to> <label> <inst> [clipFrom] [clipTo]`), `run_fetch.mjs`(오케스트레이터: 연도슬라이스 동시5 수집+병합+검증, resume), `merge_inst.mjs`, `fix_merge.mjs`(재페치 파트를 기존 합본에 union 병합 — 구멍 복구용). node_modules는 `npm install`. **출력은 `..`(=`src/`)로 나오니 수집 후 `data/`로 이동**.
  - ⚠️ **페처 버그**: `run_fetch`의 연도 슬라이스가 수집 중 크래시(exit=1)하면 그 해 일부 월이 **조용히 누락**된다(에러로 중단 안 함). 골드 2011-Q4, JPY 2011·2014가 이 케이스였음. **수집 후 반드시 큰 공백(5일↑) 스캔으로 검증**하고, 구멍은 `fetch_inst`로 해당 구간 재수집→`fix_merge`로 병합.
- **`result/`** — 모든 HTML 산출물 폴더(2026-06-19 신설). `report.html`(종합 보고서, 전 기간 2010-2026 OOS 판), `tf_level_chart.html`·`tf_level_chart_v2.html`(단계×TF×연도 회복폭, v1/v2 풀백), `cost_sensitivity.html`(비용 민감도), `fail_charts.html`(실패거래 멀티-TF 캔들 뷰어), `returns_by_year.html`·`risk_by_year.html`·`bounce_chart.html`·`baro_mfe_chart.html`. 전부 데이터 인라인 임베드(자기완결, 어디로 옮겨도 됨).
  - `report.html`은 `45_report.py`가 `seq_hours_returns.json`(39)·`cost_breakeven.json`(35)·`holding_times.json`(42)을 읽어 동적 생성. **생성 순서: 39 → 35 → 42 → 45** (전부 `cd data &&` 실행, 출력은 `../result/`). 1차 인샘플(2020-26) vs 전 기간 대비표 포함.
