2. **모든 백테스트 분석은 항상 2분·5분·10분봉 전부로 나눠서 제시한다** (한 분봉만 보여주지 말 것). 분봉마다 비용 내성·약한 해·빈도가 달라 한 개만 보면 판단이 왜곡됨.

(반복 실수 방지. 더 확실히 하려면 스크립트 상단에 `import sys; sys.stdout.reconfigure(encoding="utf-8")` 넣으면 콘솔도 UTF-8로 처리돼 자유로움.)
4. **모든 백테스트 진입 시간창 = KST 08:00 ~ 24:00(자정/밤 12시).** 신규 진입은 진입봉 KST **08:00~23:59만 허용**(00:00~07:59 컷). 자정 직전(예 23:50)에 돌파(더블비)가 떠도 그 진입은 **받아서 자연 청산까지 보유**한다 — 시간컷은 *진입*에만 적용, 보유·청산은 시간 무관. 실제 내가 매매 가능한 환경 반영. (현 스크립트 `START_H=8`이 이미 이 동작 = `khour>=8`. 4차 표도 이 기준.)

> # 🛑 최상위 정정 (2026-06-25, 4차) — 모든 헤드라인 수익은 look-ahead 산물
> **봉-OHLC 시뮬의 "한 봉 안에서 저가로 그리드 채우고 같은 봉 고가로 TP" 판정이 가짜 승리를 만들었다.** 1분봉으로 정직하게 해소하면 **전 전략·전 분봉이 적자(MDD ~95~100%)**. 아래 1차(+18.7%)·2차 per-signal(매년 +)·실거래(10m v1 +2.6%/v2 +4.3%) 헤드라인은 **전부 부풀려진 값** → 진짜는 10m v1 −28.7%/v2 −15.8%. **[4차 절](#4차-2026-06-25--look-ahead-버그-전-전략-1분봉-재검증) 먼저 읽을 것.** 옛 수치는 기록·맥락으로만 남김(삭제 안 함).

## 무엇을 하는 프로젝트인가
XAUUSD(골드) 및 FX 메이저의 **볼린저밴드 돌파 + KTR 그리드(물타기) 전략**을 Dukascopy MID 데이터로 백테스트·고도화한다. 주거래는 골드, 향후 MT4 EA를 위해 통화쌍도 검증.

## 전략 핵심 개념
- **KTR**: 세션별 변동성 단위 (아시아=2봉 범위, 유로/미장=1봉 범위). 그리드 간격의 기준.
- **신호**: BB1(종가 SMA20 ±2σ) 돌파 + BB2(시가 SMA4 ±4σ). 표준편차는 **모표준편차(÷N, Pine ta.stdev와 동일)**.
- **진입 2종**: **v1**=돌파 다음 봉 시가 즉시 진입 / **v2**=돌파 후 BB2 반대밴드 터치(풀백) 시 진입. (그 외 규칙은 v1=v2 동일)
- **그리드**: 돌파가 기준 0/−1/−2/−3/−4/−4.5 ×KTR (6차). 손절 −5×KTR.
- **익절(TP)**: 가장 깊은 체결 + 1.5×KTR, 전 포지션 일괄.
- **6차 처리**: 바닥 +1.0KTR 반등 시 탈출, 아니면 −5 풀스톱.
- 세션/DST: 미국 서머타임(3월 2째주 일요일~11월 1째주 일요일). 미장 시작 22:30/23:30 KST.

> **폴더 구조 (2026-06-18 재구성)**: `2020_2026/` → **`src/`** 로 이동. 데이터 CSV는 **`data/`** 로 분리.
> 분석 스크립트는 bare 파일명(경로 없음)을 읽으므로 **반드시 `data/`에서 실행**: `cd data && python ../src/scripts/XX.py`.

## 핵심 발견 (왜 그런지 포함)
- **랏**: 후방가중이 정답(전방 금지). 후방 1·1·2·2·3·4=생존·증거금(13단위), 강후방 1·1·2·3·5·8=평년효율(20단위)이나 추세장 꼬리위험·증거금 큼. 사다리는 "수익이 아니라 위험 모양"을 바꿈(기대 R은 거의 불변).
- **TP**: 1.5가 효율 정점. 1.8은 추세장(2025)에서 엣지 0·MDD 28% 비단조 함정. 2.0은 MDD만↑.
- **최대 체결**: 6차가 위험조정 최적. 얕게 끊으면 트레이드당 R↑이나 손절 잦아 MDD 폭발.
- **비용이 생사 결정**: 손익분기 스프레드 — 인샘플(2020-26) 2m $0.15~0.20 / 5m $0.34~0.37 / 10m $0.50~0.54. **전 기간(2010-26) 재계산은 더 빡빡: 2m $0.10~0.13 / 5m $0.21~0.25 / 10m $0.30~0.36** (가정비용 $0.30과 맞붙음). 10m만 그나마 흑자지만 비용 여유 거의 없음.
- **리스크%**: 풀스톱=자본의 2~3% 권장. 10%면 MDD 61%·경로 90%가 −50%↑ (연속손절 복리). "한 방 손실"이 아니라 "견딜 낙폭"으로 정할 것.
- **필터 전부 무용 (검증완료)**: 꼬리필터·2h추세필터·골든/데드크로스 모두 net R 감소. 돌파는 반전 이벤트라 MA추세가 후행하고, 그리드가 역추세 진입을 되돌림으로 구제하기 때문. **이 전략엔 진입 필터를 더하지 말 것.** (per-signal로 좋아 보이는 패턴 ≠ 필터로 거르면 이득)
- **운용**: 거래의 ~50%만 실제 진입(순차+새벽컷). v1은 엣지가 하루 종일 분포→시간 선택 자유, v2는 런던/저녁 집중. 평균 ~1.7건/일.
- **자본**: 수익률은 자본 크기 무관(%기반). 하한은 랏 최소단위(~$1만~1.5만). 순차라 동시보유 1개 → 증거금 부담 작음.
- **눌림 통계의 생존편향 (검증완료)**: tf_level_chart의 'N번 눌림' 칸은 *결과적으로* 그 깊이에서 멈춘 트레이드만 모은 것이라 "1번 눌리면 1R 100%"는 착시. 실제로 −1을 한 번이라도 찍은 트레이드의 **~12%(2m 13.8 / 5m 12.6 / 10m 12.3%)는 더 깊이 가서 6차 손절로 끝남**. 손실은 사라진 게 아니라 깊은(6차) 칸에 들어가 있을 뿐. 진입 시점엔 1번에서 멈출지 6차까지 갈지 모름. (`50_pullback_outcome.py`)

## 4차 (2026-06-25) — look-ahead 버그: 전 전략 1분봉 재검증
**발단**: v3(고정밴드+트레일)을 TradingView 수동 대조하다 같은봉 미래참조 발각 → 1분봉 해소(`63/64`)로 v3가 환상(+1194%·+81%)에서 적자로 무너짐. **같은 버그가 v1/v2 코어에도 있나 점검 → 있었음.**

**버그 정체** (`03c`·`39`의 `sim()`, `62`·`58` 트레일): 한 봉(2/5/10분) 안에서 **저가로 깊은 그리드를 채운 뒤 같은 봉 고가로 TP 익절**을 책정. 고가가 저가보다 먼저 왔을 수 있는데 OHLC는 봉내 경로를 모름 → "내려가 물타고 다시 올라가 익절"을 한 봉에 공짜 성사 = **가짜 승리**. 트레일판은 더 심함(매 봉 peak 재무장으로 시장가 위 SL 책정 → 폭주).

**측정** (같은 신호·랏·규칙, 해소 경로만 봉-OHLC vs 1분봉. `65`=v1 per-signal 무비용, `66`=39 실거래 풀기계):
- **per-signal 기대R(등량·무비용·TP1.5, `65`)**: 봉→1m  2m +0.200→**+0.072** / 5m +0.323→**+0.018** / 10m +0.387→**−0.045**(마이너스 전환). 승률은 ~1~3%p만 빠지지만 **얇은 엣지라 기대R이 거의 다 증발**.
- **실거래(순차+KST08~24+비용$0.30+2%복리, `66`)** — A(봉)는 39 헤드라인 정확 재현(틀 신뢰), B(1m)가 진실:

| TF·전략 | A=봉-OHLC(현 헤드라인) | **B=1분봉(정직)** |
|---|---|---|
| 10m v1 | +2.6% / MDD 63% | **−28.7% / MDD 99.6%** |
| 10m v2 | +4.3% / MDD 42.9% | **−15.8% / MDD 94.5%** |
| 5m v1 | −15.0% / 96% | −47.8% / 100% |
| 5m v2 | −11.9% / 95% | −32.3% / 99.8% |
| 2m v1 | −71.9% / 100% | −82.1% / 100% |
| 2m v2 | −53.0% / 100% | −65.5% / 100% |

- **v3(고정밴드+(c)이익권트레일, 3그리드, `64` 1분봉+시간필터 08~24, N best=0.5)**: 10m −35.6% / 5m −63.6% / 2m −96.3%, **전부 MDD ~100%**. 시간필터 넣어도 동일(10m −38.5→−35.6% 미세). → v3도 파산.
- **결론**: 유일 흑자였던 10m v1/v2가 **−28.7%·−15.8%로 뒤집히고 MDD ~95~100%로 폭발**. look-ahead가 10m을 흑자로 보이게 한 유일한 이유. **정직한 1분봉 + 08~24 진입창에선 v1·v2·v3 × 2/5/10분 = 9개 전부 적자(MDD 95~100%, 계좌 파산).** 10m이 셋 다 "가장 덜 나쁨"이나 여전히 −16~−36%. 2차의 "per-signal 매년 + → OOS 유지" 기둥도 봉-OHLC 산물.
- **교훈**: 봉 OHLC로 그리드+TP/트레일을 같은 봉에 해소하면 무조건 가짜 승리. **봉내 동시 이벤트(체결·TP·손절)는 반드시 1분봉(또는 더 미세)으로 시간순 해소**해야 함. 앞으로 모든 손익 판정은 1분봉 해소(`66` 틀)로.
- 영향받는 옛 수치(전부 부풀려짐, 기록용): 1차 표(10m v1 +18.7%), 2차 per-signal·실거래 OOS 표, 3차 트레일/`53`/`56`(B) 도달·놓친수익(같은봉 fill+TP 포함). **`56 breakout_reach.html`도 봉-OHLC라 낙관** → 재판정 시 1분봉 필요.

## 5차 (2026-06-25) — 더캔이지추격깨(수달 재량법) 1H 방향게이트 (`67`, `result/dckj_gate.html`)
PDF `더캔이지추격깨정의.pdf`(네이버카페 수달 정의편) 기반. 매 1H 마감마다 4요소 가중투표(+ =LONG)로 방향 우위 산출 → 하위봉(2/5/10m) 돌파신호 방향이 bias와 일치할 때만 진입. 미래참조 차단(진행중 1H봉 미사용). 해소는 `66` 1분봉 정직틀.
