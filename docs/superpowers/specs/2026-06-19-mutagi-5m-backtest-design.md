근거 명세서: `새로운명세/무따기_통합_백테스팅_명세서.md` §4.1, §8

## 1. 목적 / 범위

명세서 §8이 1순위로 지정한 **5분봉 Tip(§4.1)** 계열을, 사용자가 정의한
**3개 변형 전략**으로 골드에 백테스트한다. 규칙이 가장 명확한 부분만 떼어
독립 검증하는 것이 목표이며, 전략 A/B·더캔이지추격깨패 방향판단·구간분할·
트레일링 등 나머지 시스템은 **이번 범위에서 제외**한다.

핵심 질문: "20/120 SMA 크로스를 추세 필터로 쓰고, 진입 트리거만 달리한
3가지 단순 룰이 골드에서 연도별로 어떤 성과를 내는가?"

## 2. 공통 정의

| 항목 | 정의 |
|---|---|
| 데이터 | `xauusd_{2m,5m,10m}_2010-01-01_2026-06-16.csv` (골드만). 컬럼 = epoch(KST 환산), O, H, L, C |
| 추세 이평 | 20 SMA, 120 SMA (종가 기준, 단순이동평균) |
| 골든크로스 | `sma20[i] > sma120[i]` AND `sma20[i-1] <= sma120[i-1]` (봉마감 확정) |
| 데드크로스 | `sma20[i] < sma120[i]` AND `sma20[i-1] >= sma120[i-1]` (봉마감 확정) |
| 4/4 시가 볼린저 | 시가(open) SMA4 ± 4σ. 표준편차는 **모표준편차(÷N, Pine ta.stdev / 레포 BB2와 동일)** |
| 진입 타이밍 | 트리거 봉마감으로 확정 → **다음 봉 시가(open)** 진입 |
| 청산 타이밍 | 반대 크로스 확정 봉의 **다음 봉 시가(open)** 청산 |
| 동시 포지션 | 1크로스당 진입 1회, 청산까지 보유 → 자연히 동시 1포지션 |
| 시간 필터 | 없음 (지표/미장 회피 OFF. 추후 토글 후보) |
| 미청산 처리 | 데이터 끝에 포지션이 열려 있으면 마지막 종가로 청산하고 `open_at_end` 플래그 |

분봉(2m·5m·10m)마다 동일 로직을 독립 실행한다(CLAUDE.md 규칙 #2 준수).
5m이 명세서가 지정한 주력 TF, 2m·10m은 비용내성·빈도 비교용.

## 3. 3개 전략 (LONG 기준 정의, SHORT는 완전 대칭)

직전 골든크로스 이후 **첫 번째** 조건 충족 봉에서만 트리거가 발동한다.
이후 데드크로스로 청산되며, 그 전에는 추가 진입 없음.

| 전략 | LONG 진입 트리거 (골든크로스 이후 첫 …) | 청산 |
|---|---|---|
| **S1 원비터치** | `low <= 4/4 시가 하단밴드` 인 봉 → 다음 봉 시가 | 데드크로스 |
| **S2 즉시** | (트리거 없음) 골든크로스 확정 다음 봉 시가 | 데드크로스 |
| **S3 장기이평터치** | `low <= sma120` 인 봉 → 다음 봉 시가 | 데드크로스 |

비교 연산자는 **`<=`** 로 통일.

### SHORT 대칭
- 국면: 골든크로스 → **데드크로스** 로 치환, 청산은 **골든크로스**.
- S1: `high >= 4/4 시가 상단밴드`
- S2: 데드크로스 확정 다음 봉 시가
- S3: `high >= sma120`

### 엣지 케이스
- 골든크로스 후 트리거가 발동하기 전에 데드크로스가 오면(특히 S1/S3) → 그 국면은 무진입.
- 충분한 워밍업 전(`i < 120`, 또는 밴드/SMA 미산출 구간)은 신호 무시.

## 4. 손익 계산

- **포지션 크기**: 고정 1랏(1 contract). 수익률 비교는 자본 무관이므로 절대/비율 둘 다 산출.
- **포인트 손익(gross)**: LONG = `exit - entry`, SHORT = `entry - exit`.
- **비용(net)**: 라운드트립 **0.4** 차감 (`net = gross - 0.4`). 그로스도 함께 보관.
- **% 수익률**: `포인트 / entry * 100` (gross·net 각각).
- **보유봉수**: 진입봉~청산봉 사이 봉 개수.

### 연도별 집계(전략 × 분봉 × 방향마다)
연도는 **진입 시점(KST) 연도** 기준. 각 셀:
- 트레이드 수
- 승률(net 기준)
- 평균 / 총 포인트 (gross·net)
- 평균 / 총 % (gross·net)
- 손익비 PF = 총이익 / |총손실| (net)
- 누적 net 포인트 곡선의 최대낙폭(MDD)
- 평균 보유봉수

전체 기간(2010–2026) 합계 행도 포함.

## 5. 산출물 / 구조 (2 스크립트)

기존 레포의 `signals → report` 패턴을 따른다. 표준 라이브러리만 사용.

### 5.1 `src/scripts/60_mutagi_5m_signals.py` — 엔진
- 입력: `xauusd_{2m,5m,10m}_...csv`
- 처리: SMA20/120, 4/4 시가 볼린저, 크로스 탐지, 3전략 × 2방향 트레이드 생성.
- 출력: `mutagi_trades_{tf}.csv` (data/) — 한 행 = 한 트레이드.
  컬럼: `strategy, direction, tf, entry_dt_kst, entry_epoch, exit_dt_kst, entry_price, exit_price, points_gross, points_net, pct_gross, pct_net, hold_bars, year, open_at_end`
- 콘솔 출력은 **ASCII만** (CLAUDE.md 규칙 #3). 전략별 트레이드 수 요약 출력.

### 5.2 `src/scripts/61_mutagi_report.py` — 리포트
- 입력: `mutagi_trades_{2m,5m,10m}.csv`
- 출력: `result/mutagi_5m_report.html` (데이터 인라인 임베드, 자기완결).
- 구성: 전략(S1/S2/S3) × 분봉(2m/5m/10m) × 방향(롱/숏) 매트릭스 +
  연도별 표(드롭다운 또는 섹션). 헤드라인은 5m 우선 노출.

### 실행
```
cd data && python ../src/scripts/60_mutagi_5m_signals.py
cd data && python ../src/scripts/61_mutagi_report.py
```
(bare 파일명 입출력이라 cwd가 `data/`여야 함. HTML은 `../result/`로 출력.)

## 6. 명세서와의 차이 / 의도적 단순화

- 명세서 §4.1 원문은 "20 SMA 돌파 + 크로스 + 첫 원비/더블비 봉마감"의 복합 조건이나,
  본 백테스트는 **크로스를 추세 필터로, 진입 트리거를 3가지로 분리**해 변형 검증한다(사용자 정의).
- 원비/더블비의 정밀 정의(밴드 재터치 등)는 미사용 — S1은 "밴드 1회 터치(`<=`)"로 근사.
- KTR 손절·익절·구간분할·트레일링 미적용(이 3전략은 데드크로스 청산 단일 규칙).
- 시간대/지표 회피 필터 미적용.

이 단순화들은 모두 추후 파라미터/토글로 확장 가능하나 **이번 범위 밖**.

## 7. 검증 포인트 (미래참조 편향 주의)

- 크로스·밴드·SMA는 모두 **봉마감 확정값**만 사용하고, 진입은 **다음 봉 시가** → 미래참조 없음.
- S1/S3 트리거 봉의 판정에 그 봉의 종가/저가는 봉마감 후 확정값이므로 OK
  (진입은 그 다음 봉 시가라 같은 봉 내 미래참조 아님).
